## Purpose
Define shared terminology to ensure cross-repo consistency.

## Scope
- Covers: terms required for consistent documentation.
- Does not cover: implementation-specific jargon.

## Table of Contents
{:toc}

## Terms
- Defaults-first: a rule that mandatory defaults are applied before any optional configuration.
- Diagnostics Session: a single lifecycle of diagnostics data for one operation.
- Runner: a component that executes a pipeline with deterministic ordering.
- Strategy: a deterministic policy used by a Runner for a specific behavior.
